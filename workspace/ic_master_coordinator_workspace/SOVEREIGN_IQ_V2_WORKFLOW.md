# Sovereign-IQ V2.0 工作流定义
## 2026 AI 智能投委会（IC）高效协同工作流

---

## 阶段总览

| 阶段 | 名称 | 时间节点 | 参与角色 |
|------|------|----------|----------|
| 一 | 项目准入与合规网关 | T+0 min | 秘书、风控委员 |
| 二 | 分布式专家并发研判 | T+5 ~ T+30 min | 财务、行业、战略、法务 |
| 三 | 逻辑自愈与交叉验证 | 触发制 | 秘书、财务、法务 |
| 四 | 红蓝对抗与逻辑压榨 | T+30 ~ T+50 min | 主席、全体专家 |
| 五 | 决策合成与工程化留痕 | T+50 ~ T+60 min | 主席、秘书 |

---

## 阶段一：项目准入与合规网关（T+0 min）

### 触发条件
项目资料提交至秘书 Agent

### 执行步骤

**1.1 全量接收与分发**
- 秘书 Agent 接收标的项目资料（BP、尽调底稿、财务报表等）
- 根据 TaskGraph 协议自动在各专家 Workspace 间建立逻辑关联
- 生成项目任务树，分配唯一项目 ID

**1.2 物理隔离加载**
- 系统自动调用各专家预挂载的 Milvus 知识库
- 实现"知识主权隔离"，防止语义交叉干扰
- 知识库映射：
  - 财务 Agent → `milvus_financial_benchmark_db`
  - 法务 Agent → `milvus_law_rules_db`
  - 行业 Agent → `milvus_industry_whitepapers_db`
  - 战略 Agent → `milvus_policy_db`
  - 风控 Agent → `milvus_internal_sop_db`
  - 主席 Agent → `milvus_sop_case_db`

**1.3 合规性预筛**
- 风控 Agent 立即调取预挂载的 AML 规则库
- 对项目及关联方执行"一键负面清单核查"
- 输出：风险预警信号（有/无）+ KYC/AML 初审报告

### 准入标准
- AML 核查通过 → 进入阶段二
- 核查未通过 → 触发快速拒绝流程，输出拒绝理由书

### 输出产物
- `project_id`: 项目唯一标识
- `task_graph`: 任务拓扑关系图
- `aml_kyc_report`: KYC/AML 合规初审报告
- `risk_signal`: 风险预警信号（0/1）

## 阶段二：分布式专家并发研判（T+5 min - T+30 min）

### 触发条件
阶段一准入通过

### 并发执行（4 个专家 Agent 同时启动）

#### 2.1 财务价值锚定（财务 Agent）

**核心任务：** PB-ROE 修正算法 + DCF 压力测试

**估值公式：**
```
V = (ROE - g) / (r - g)
```
其中：
- r = 动态折现率（结合宏观模型修正）
- g = 长期增长率
- ROE = 净资产收益率

**执行内容：**
- 调用 `wilcox_valuation_engine()` 计算估值带
- 调用 `dcf_stress_test()` 执行多维度现金流压力测试
- 对比预挂载行业财报标杆库，输出横向对标报告
- 实时调取 A股/港股最新财报数据

**输出：** `valuation_band`（估值区间）、`dcf_stress_report`

#### 2.2 技术壁垒穿透（行业 Agent）

**核心任务：** 专利密度分析 + 供应链韧性评估

**执行内容：**
- 调用 `analyze_patent_density()` 解析全球专利数据库
- 评估技术自研深度与专利壁垒强度
- 调用 `supply_chain_stress_test()` 模拟技术断供场景
- 评估企业在"反内卷"政策下的价格毛利稳定性

**输出：** `patent_density_report`、`supply_chain_resilience_score`

#### 2.3 战略对齐模拟（战略 Agent）

**核心任务：** 政策匹配度 + 终值敏感性分析

**执行内容：**
- 调用 `strategic_alignment_check()` 将项目与"十四五"规划语义对齐
- 调用 `policy_impact_simulator()` 模拟医保集采、产业补贴变动对 IRR 的影响
- 评估国资 LP 出资比例稳定性
- 输出政策契合度评分（0-100）

**输出：** `policy_alignment_score`、`irr_impact_matrix`

#### 2.4 法律无损穿透（法务 Agent）

**核心任务：** 超长上下文尽调底稿扫描 + 合规审查

**执行环境：** NVIDIA OpenShell 物理隔离沙箱

**执行内容：**
- 调用 `penetrate_legal_docs()` 扫描数千页合同与尽调底稿
- 穿透复杂关联交易与隐秘持股
- 调用 `check_compliance_2025()` 比对最新监管通告库
- 确保处理流程符合《网络数据安全管理条例》

**输出：** `legal_risk_report`、`compliance_gap_analysis`

### 并发同步机制
- 秘书 Agent 通过 TaskGraph 协议协调 4 个专家 Agent 并发执行
- 各 Agent 独立运行，互不阻塞
- 任务状态实时上报秘书 Agent

## 阶段三：逻辑自愈与交叉验证（Self-Correction Loop）

### 触发条件
财务 Agent 发现以下异常信号之一：
- 应收账款增速远超收入增速
- 毛利率异常波动
- 现金流与净利润严重背离
- 资产负债表与现金流量表勾稽失衡

### 执行步骤

**3.1 自愈触发**
- 财务 Agent 向秘书 Agent 发送异常信号 + 异常类型标签
- 秘书 Agent 激活自愈任务流，自动扩展任务图

**3.2 二次深度扫描**
- 法务 Agent 接收自愈请求
- 对原始交易凭证 OCR 扫描件执行深度穿透
- 寻找可能遗漏的虚假陈述证据
- 调用 `penetrate_legal_docs()` 复查关联交易路径

**3.3 反馈闭环**
- 法务 Agent 将穿透结果回传秘书 Agent
- 秘书 Agent 判断是否需要追加其他专家研判
- 若异常确认，标记高风险项至阶段四主席 Agent

### 输出产物
- `self_correction_report`: 自愈扫描报告
- `risk_escalation_flag`: 风险升级标记（有/无）
- `additional_findings`: 额外发现清单

## 阶段四：红蓝对抗与逻辑压榨（T+30 min - T+50 min）

### 触发条件
阶段二所有专家 Agent 完成初步研判

### 执行步骤

**4.1 对抗辩论机制**
- 主席 Agent 识别各专家意见趋同程度
- 若存在锚定效应，强制开启"红队模式"
- 各专家 Agent 须搜索对冲性负面证据

**4.2 黑天鹅预演（必选）**
- 行业 Agent：提出 3-5 个技术断供/供应链断裂极端场景
- 战略 Agent：提出 3-5 个政策突变极端场景（集采叫停、地缘冲突等）
- 法务 Agent：提出 3-5 个监管收紧或诉讼爆发场景
- 财务 Agent：提出 3-5 个流动性危机或估值泡沫场景

**4.3 逻辑闭环审计**
- 主席 Agent 对所有对抗性结论进行最终审计
- 强制要求负面证据覆盖所有关键假设
- 输出：`adversarial_debate_summary`

### 输出产物
- `adversarial_findings`: 对抗性发现清单
- `black_swan_scenarios`: 黑天鹅场景列表（每个 Agent ≥3 个）
- `assumption_stress_results`: 关键假设压力测试结果

---

## 阶段五：决策合成与工程化留痕（T+50 min - T+60 min）

### 触发条件
阶段四对抗辩论完成

### 执行步骤

**5.1 权重化评分**
- 主席 Agent 根据赛道类型自动调整专家权重

| 赛道类型 | 权重调整策略 |
|----------|-------------|
| 硬科技（半导体、AI） | 行业专家权重 +20% |
| 生物医药 | 战略专家权重 +15%，行业专家权重 +10% |
| 消费 | 财务专家权重 +15% |
| 国资主导 | 战略专家权重 +20% |

**5.2 投决书生成**
- 主席 Agent 汇总所有阶段结论
- 调用 `synthesize_ic_report()` 生成投决建议书
- 建议书结构：
  1. 项目概述
  2. 专家研判摘要（按权重排序）
  3. 估值区间与核心假设
  4. 风险清单（含黑天鹅场景）
  5. 对抗性辩论结论
  6. 投决建议（同意/有条件同意/拒绝）
  7. 后续跟踪指标

**5.3 原生审计日志**
- 秘书 Agent 整理全流程原生日志
- 确保决策过程 100% 可审计、可追溯
- 日志包含：每个 Agent 的输入、输出、耗时、调用工具列表

### 输出产物
- `investment_recommendation_report`: 投决建议书（PDF/HTML格式）
- `audit_log`: 原生决策链路日志（满足2026年监管要求）
- `decision_metadata`: 决策元数据（时间戳、参与 Agent、结论摘要）

## 工作流状态机

```
[项目提交] 
    ↓
[阶段一：准入筛查] → AML未通过 → [快速拒绝]
    ↓ 通过
[阶段二：并发研判]（财务/行业/战略/法务 并行）
    ↓ 全员完成
[阶段三：自愈验证]（条件触发）
    ↓ 或跳过
[阶段四：红蓝对抗]
    ↓
[阶段五：投决合成]
    ↓
[输出：建议书 + 审计日志]
```

---

## 时间约束

| 阶段 | 最大耗时 |
|------|----------|
| 阶段一 | 5 分钟 |
| 阶段二 | 25 分钟 |
| 阶段三 | 10 分钟（并行，不额外增加总时间） |
| 阶段四 | 20 分钟 |
| 阶段五 | 10 分钟 |
| **总计** | **≤60 分钟** |

---

## 关键成功指标（KPI）

- 自愈触发准确率：>90%（财务异常→法务穿透确认率）
- 对抗覆盖率：每个关键假设至少 1 条负面证据
- 审计日志完整率：100%（每个 Agent 操作留痕）
- 投决书产出时效：≤60 分钟
